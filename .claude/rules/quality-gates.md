# Quality Gates — Layer Trust Architecture

> Адаптировано из quality-system-plan.md (PMBOK 7 / ISO 25010)

## Слои доверия

Каждый слой проверяется **только относительно предыдущего**. Никогда — сам себя.

```
L0: Источник истины      → Pydantic models, OpenAPI spec, JSON Schema
L1: Код приложения       → unit-тесты проверяют L1 против L0
L2: Интеграции           → integration-тесты проверяют L2 против L1 (реальная БД!)
L3: UI (Mini App)        → e2e-тесты проверяют L3 против L2
L4: Runtime              → логи, мониторинг, health checks
```

## Уровни готовности

| Уровень | Что значит | Проверка |
|---------|-----------|----------|
| L1 🔧 | Код написан | Файлы созданы |
| L2 ✅ | Тест OK | pytest пройден |
| L3 🚀 | На сервере | systemctl active |
| L3.5 🔍 | Верифицирован | journalctl чисто через 5 мин |
| L4 ✅ | В проде | Работает 24+ часа |

**ЗАПРЕЩЕНО:** L3 → L4 без L3.5 (проверка логов обязательна)

## 5 паттернов качества

### A: Каскадная верификация
Баг найден → `grep` по проекту → исправить ВСЕ вхождения за один проход.

### B: Двойная проверка (Dual-Source)
Один факт — два способа верификации. Расхождение = красный флаг.
Unit-test + integration-test на один эндпоинт.

### C: Hash-gating
Snapshot-тесты API-ответов, schema validation для миграций БД.

### D: Реестр исключений
`auditors/known_exceptions.yaml` — quirks Max API, подавление ложных срабатываний.

### E: Полнота проверки
100% покрытие критических путей обязательно.

## Анти-паттерны

| # | Запрещено | Почему |
|---|-----------|--------|
| 1 | Аудитор проверяет сам себя | Circular reasoning |
| 2 | Только happy path | Баги в edge cases |
| 3 | Моки вместо реальной БД | Скрывают рассинхронизацию (20-50% ошибок) |
| 4 | Один гигантский тест-сьют | Разбить: unit → integration → e2e |
| 5 | Фикс без grep по проекту | 1 ошибка исправлена, 8 остались |

## CHECK-1: До кода (задача > 50 строк)
- [ ] MVP = 1 кейс
- [ ] Готовое решение существует?
- [ ] Путь реалистичен?

## CHECK-2: После кода, до коммита (файлов >= 3)
- [ ] pytest OK?
- [ ] Нет хардкод путей?
- [ ] Secrets не в коде?

## CHECK-3: После деплоя на VPS
- [ ] `journalctl -u dom-max -n 20` — нет ERROR?
- [ ] Cron/systemd расписание верное?
